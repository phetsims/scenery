Overall Code Structure

Overall Code structure

Note: a11y is a synonym for accessibility.

The DAG features of the a11y side of Scenery are handled parallel to Nodes in Scenery. Each Node with the Parallel DOM trait added to its prototype has N PDOMInstances based on the number of times it has been added to the scene graph. The PDOM elements of each Node are created and handled with PDOMPeer. There is a 1x1 relationship of PDOMInstance and PDOMPeer.

The PDOMPeer is the type that manages all HTML Elements. The Node is then thought to be "represented" in the PDOM by the PDOMPeer's HTML Elements.
